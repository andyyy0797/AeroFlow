# Efficient Extension for AeroFlow (Task 2)

## 1) Project Introduction

Task 2 is a terminal-based demo program built on AeroFlow mock booking data.
It demonstrates:

- **Data Structure:** Trie (for fast email prefix search)
- **Algorithm:** Counting Sort (for integer price sorting)

The program reads `mockBookings.json`, builds an email Trie index, and provides an interactive menu.

## 2) Project Structure

- `main.py` — Main program logic (Trie, Count Sort, menu, Rich UI)
- `tools/mockData.py` — Script to generate mock data with specified quantity
- `mockBookings.json` — 50,000 mock booking records generated by `mockData.py`

## 3) Core Features

### 3.1 Search Booking by Email (By Trie)

- Builds Trie index using `user_email`
- Supports email prefix search
- Results shown in Rich table
- Shows key booking fields:
  - Booking ID
  - Email
  - Destination
  - Departure
  - Class
  - Total Price
  - Sub passenger count
- Pagination:
  - 10 records per page
  - Left/Right arrow keys to change pages
  - Shows current page and total pages

### 3.2 Run Demo (By Trie + Count Sort)

- Uses sample prefixes (for example: default prefix, `ma`, `zz`)
- For each prefix, shows:
  - Prefix summary (matched records, cheapest, highest)
  - Top 3 cheapest bookings (sorted by `totalPrice`)
- Layout uses fixed two-column view:
  - Left: Prefix Result
  - Right: Top 3 Cheapest
- If no matched records, layout remains aligned and stable (`N/A` and placeholder rows)

### 3.3 Sort All Orders by Total Price (By Count Sort)

- Sorts all records by `totalPrice` with Counting Sort
- Shows summary panel:
  - Total sorted records
  - Lowest price
  - Highest price
  - Average price
- Shows two ranking tables:
  - Top 10 cheapest
  - Top 10 highest

## 4) How It Works

### 4.1 Startup Flow

When the program starts:

1. Find `mockBookings.json`
2. Read JSON data
3. Create a Trie with index on `user_email`
4. Ready to use

### 4.2 Trie Design

- `TrieNode` stores:
  - `children`
  - `isEnd`
  - `records` (bookings ending at that node)
- `Trie.findByPrefix(prefix)`:
  - Iterate through prefix string
  - Collects all records under matched subtree

### 4.3 Counting Sort Design

- Counting Sort is used on integer key `totalPrice`
- Time complexity: **O(n + k)**
  - `n` = number of records
  - `k` = price range (`max - min + 1`)

## 5) How to Execute?

Please make sure that you are located in the `Task_2` folder and have `uv` installed, if not, execute the following command:

```bash
pip install uv
```

Step 0 (Optional): Execute this command to create another 50,000 demo booking records:

```bash
uv run python tools/mockData.py
```

Step 1: Execute this command to install all required dependencies with `uv`:

```bash
uv sync
```

Step 2: Execute this command to start the application:

```bash
uv run python main.py
```
